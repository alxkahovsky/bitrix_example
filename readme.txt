Задание №1 файл main.py
В коде не использовал API IsDayoff(), тк явно не указываются государственные праздники;
В коде используется файл конфигурации, его необходимо дополнить перед запуском. Пример файла config.cfg:
-------------------------------------------------------------------------------------------------------
[BITRIX_API]                                                                                          |
CALL_URL=https://введите вебхук для вызова rest api/                                                  |
-------------------------------------------------------------------------------------------------------
В боевом окружении самый простой способ запускать данный скрипт кроном (linux) или через планировщик задач (wwindows)
еждневно;
В коде присутствует условие, что бы не создавалась запись не для праздников идущих подряд (это мое дополнение);

Задание №2 файл simple_server.py (апускать в терминале python simple_server.py)
В коде используется файл конфигурации, его необходимо дополнить перед запуском. Пример файла config.cfg:
-------------------------------------------------------------------------------------------------------
[BITRIX_API]                                                                                          |
CALL_URL=https://введите вебхук для вызова rest api/                                                  |
-------------------------------------------------------------------------------------------------------
Для релизации функционала используется простой http сервер, запросы отправлял Postman'ном
Мои замечания/дополнения к алгоритму:
1. Не предусмотрена валидация номеров телефона, адресов. В идеале я бы предпочел получать данные которые уже готовы
к потправке в Битрикс;
2. Не обрабатывается список товаров. Поскольку БД товаров у меня нет я передаю данный список строкой в user_field
сделки;
3. Товары в теле JSON'a я бы передавал списком словарей 'products': [{'product_id': 1, 'product_name': 'Some product'},]
искать что то в БД по строковому зназванию, мне кажется плохой идеей.
4. Не передается количество товаров;

Docker не использовал. Для понимания: мои познания докера ограничены прочтением документации по данному продукту, я
понимаю для чего используется, но опыта использования в боевом проекте у меня нет.
